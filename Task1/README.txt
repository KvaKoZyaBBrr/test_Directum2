Вариант 1

Таблицы Course и Student - справочники курсов и студнтов соответственно. Ключ - ID
Таблица Schedule для создания расписания с временем и количеством участников. Ключ - поля IDCourse и time (курсы могут проводиться несколько раз)
Таблица Lessons - хранение участников на курсе. Ключ - поля IDStudent и IDCourse (студент не может посещать один и тот же курс, но может посещать разные курсы). Поле time - часть внешнего ключа, поэтому его тоже надо добавить для консистентности
Требование по добавлению студентов под количетсов мест на курсе не придумал как сделать кроме как программно, либо тригером при добавлении записи в Lessons с проверкой по количеству из Schedule.
